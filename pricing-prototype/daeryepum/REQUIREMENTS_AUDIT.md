## 3. 매출 예측 모델 사양

### 회귀 모델

```
답례품 일매출 = 23,161원 × 청첩장 일주문건수(8주 전) − 1,126,839원
```

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| 계수(β) | 23,161원 | 청첩장 1건당 답례품 매출 기여도 |
| 절편(α) | −1,126,839원 | 기저 오프셋 |
| Lag | 56일 (8주) | 최적 시차 (r=0.472) |
| R² | 0.223 | 설명력 22% |
| 부차 피크 | 3주 (r=0.333) | "미리 준비형" 주문자 |

### 모델 한계

- 답례품 매출 변동의 **22%만** 청첩장 주문으로 설명
- 나머지 **78%는** 시즌성, 마케팅, 검색/직접 유입, 위탁 등 타 변수
- 현재 청첩장 일 120~140건 수준 → 모델 예측 일매출 약 150~200만원
- 실제 일평균 약 270만원 → 크로스셀은 "기저 매출 확보" 수단

### 예측 산출 방식

1. 향후 12주 각 주차에 대해
2. 해당 주의 **8주 전** 청첩장 일별 주문수를 조회
3. 회귀식으로 일매출 산출 → ×7 = 주간 매출
4. 8주 전 데이터 유무에 따라 신뢰도(높음/보통/데이터없음) 표시

## 4. 데이터 소스 매핑

### 부가상품 단독주문 (3xxxxxx)

```
CUSTOM_ETC_ORDER (주문)
  └→ CUSTOM_ETC_ORDER_ITEM (주문항목, card_seq → S2_Card)
  └→ CUSTOM_ETC_ORDER_WeddInfo (예식정보)
```

- 배송정보: `recv_name`, `recv_hphone`, `recv_address`, `recv_address_detail`, `recv_msg`
- 결제금액: `card_sale_price` (총액, 단가×수량 포함)

### 청첩장+답례품 주문 (4xxxxxx)

```
custom_order (주문)
  └→ custom_order_item (주문항목, card_seq → S2_Card)
  └→ DELIVERY_INFO (배송정보)
  └→ custom_order_WeddInfo (예식정보)
```

- 배송정보: `DELIVERY_INFO.NAME`, `HPHONE`, `ADDR`, `ADDR_DETAIL`, `DELIVERY_MEMO`
- 결제금액: `item_sale_price × item_count` (단가×수량 계산 필요)

### 공통 필터

- 답례품 카테고리: `S2_Card.Card_Div = 'D01'`
- 유효 주문: `status_seq >= 1`
- 성능: 모든 쿼리에 `WITH (NOLOCK)` 힌트
